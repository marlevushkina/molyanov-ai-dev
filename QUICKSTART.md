# Быстрый старт: Как работать с этим фреймворком

Этот фреймворк помогает делать проекты при помощи AI-агентов. Вы описываете задачу — система сама планирует, кодит, проверяет и доводит до результата.

## Что нужно для начала

1. Подписка Claude Code (MAX или Pro)
2. Установленный Git и GitHub CLI (`gh`)
3. Скопировать фреймворк к себе (см. "Установка" ниже)

## Установка (один раз)

```bash
git clone https://github.com/marlevushkina/molyanov-ai-dev.git /tmp/molyanov-ai-dev
cp -rp /tmp/molyanov-ai-dev/commands ~/.claude/commands
cp -rp /tmp/molyanov-ai-dev/agents ~/.claude/agents
cp -rp /tmp/molyanov-ai-dev/skills ~/.claude/skills
cp -rp /tmp/molyanov-ai-dev/shared ~/.claude/shared
cp -rp /tmp/molyanov-ai-dev/hooks ~/.claude/hooks
chmod +x ~/.claude/hooks/*.sh ~/.claude/shared/scripts/*.sh
```

После этого команды доступны в любом проекте.

---

## Главный маршрут: от идеи до результата

Запомните 6 шагов. Система сама подскажет следующий после каждого.

```
Шаг 1  /init-project             Создать проект (5 мин)
           |
Шаг 2  /new-user-spec            Описать что хотите (10-15 мин)
           |                      Система задаёт вопросы — вы отвечаете
           |
Шаг 3  /new-tech-spec            Технический план (автоматически)
           |                      Вы только говорите "ок" или "нет"
           |
Шаг 4  /decompose-tech-spec      Разбивка на задачи (автоматически)
           |
Шаг 5  /do-feature               Выполнение (автономно, без вас)
           |                      Или /do-task — одна задача за раз
           |
Шаг 6  /done                     Финализация
```

В конце любой сессии: `/save-progress` — сохранит, где остановились.

---

## Когда какой шаг использовать

### "Хочу новый проект с нуля"
Начинайте с Шага 1: `/init-project`

### "Проект уже есть, хочу добавить фичу"
Начинайте с Шага 2: `/new-user-spec`

### "План уже есть, хочу чтобы сделали"
Начинайте с Шага 5: `/do-feature` или `/do-task`

### "Нужно быстро починить баг"
Не нужен фреймворк. Просто скажите Claude что починить.

### "Заканчиваю работу на сегодня"
`/save-progress` — запишет всё в лог. Завтра Claude вспомнит.

## Подсказки для каждого шага

### /init-project
**Что от вас нужно:** название проекта, больше ничего.
**Что получите:** готовую структуру папок + git-репо.
**Следующий шаг:** `/init-project-knowledge` или сразу `/new-user-spec`

### /new-user-spec
**Что от вас нужно:** отвечать на вопросы агента-интервьюера простыми словами. Не нужно технических терминов — говорите как есть.
**Что получите:** документ-спецификацию на понятном языке.
**Следующий шаг:** `/new-tech-spec`

### /new-tech-spec
**Что от вас нужно:** прочитать план и сказать "ок" или "нет, давай иначе".
**Что получите:** технический план, проверенный несколькими агентами.
**Следующий шаг:** `/decompose-tech-spec`

### /decompose-tech-spec
**Что от вас нужно:** ничего, одна команда.
**Что получите:** набор конкретных задач с чеклистами.
**Следующий шаг:** `/do-feature` (всё автономно) или `/do-task` (по одной задаче)

### /do-feature
**Что от вас нужно:** ничего. Можно уйти.
**Что получите:** готовый код, прошедший ревью.
**Следующий шаг:** проверьте результат, затем `/done`

### /do-task
**Что от вас нужно:** указать номер задачи.
**Что получите:** выполненную задачу с ревью.
**Следующий шаг:** `/do-task` (следующая задача) или `/save-progress` (если на сегодня хватит)

### /save-progress
**Что от вас нужно:** подтвердить лог-запись.
**Что получите:** зафиксированное состояние проекта.
**Следующий шаг:** завтра просто откройте проект — Claude прочитает LOG.md и продолжит.

### /done
**Что от вас нужно:** указать какую фичу завершить.
**Что получите:** обновлённую документацию, архив фичи.
**Следующий шаг:** `/new-user-spec` для следующей фичи.

## Частые вопросы

**"А если задача простая — скрипт на 10 строк?"**
Не нужен полный цикл. Просто скажите Claude что сделать. Фреймворк — для задач от нескольких часов и больше.

**"Что если агент сделал не то?"**
Скажите что не так. Система исправит. Если совсем не то — `/save-progress`, начните свежую сессию.

**"Сколько это стоит в токенах?"**
Полный цикл /do-feature с ревью — примерно 30-50% дневного лимита MAX. Для простых задач через /do-task — значительно меньше.

**"Можно ли пропустить шаги?"**
Да. Если у вас уже есть план — начинайте с /decompose-tech-spec. Если задачи уже есть — с /do-task.
